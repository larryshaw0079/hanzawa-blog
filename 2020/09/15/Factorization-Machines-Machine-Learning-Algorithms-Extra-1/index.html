Introduction

对于一般的线性模型或者广义线性模型，我们的模型是\(\hat y = w_0 + \sum_{i=1}^n w_i x_i\)或者\(g^{-1}(\hat y)=w_0+\sum_{i=1}^n w_i x_i\)的形式。在推荐的场景下，我们希望考虑特征之间的关系，比如特征“男性”往往和“运动”之类的特征有关系，实践中这种特征组合往往能带来很好的模型效果提升。由于推荐场景下特征往往是Categorical的，在One-hot编码之后有很高的稀疏性，这会为我们训练考虑特征组合的模型带来很大的问题（后面会说明）。

Factorization Machine

Factorization Machine （简称FM），是一种

blog

Feature Crossing

考虑特征组合的初始模型 \[ \hat y(x) = w_0 + \sum_{i=1}^n w_i x_i + \sum_{i=1}^{n-1} \sum_{j=i+1}^n w_{ij} x_i x_j \] 不过这会有一些问题，比如\(x_i\)是用户B，\(x_j\)是电影TI，而我们想知道用户B和电影TI之间的关系，而训练集中不存在\(x_i=1\)且\(x_j=1\)的样本让我们学习，这样会导致\(w_{ij}\)永远等于\(0\)，但实际上用户B和电影TI之间的关系可能是存在的，我们只是没有训练样本来推断它而已。

FM的做法是对于这种\(w_{ij}\)，我们想办法去估计它。首先对于每个特征\(x_i\)，FM假设其存在一个\(k\)维的隐向量\(\mathbf v_i\)与之对应（也可以理解为特征的一个embedding）。如下图展示了用户特征与电影特征之间的特征组合（\(k=2\)）：

任意两个特征之间的关系可以通过隐向量之间的内积来体现，如果两个特征之间相关度越高，那么他们之间的内积也会越大。不过FM是如何解决刚才所说的，用户B和电影TI之间的关系没有样本来体现，我们要如何学习用户B和电影TI这两个特征之间的关系的问题呢？大家可以想想，虽然没有用户B和电影TI都为\(1\)的样本，但这两个特征对应的隐向量是不为\(0\)的，我们依然可以通过两个向量的内积来反应用户B和电影TI的相关性（这里其实和自监督对比学习很相似😄）。

于是我们的模型变为： \[ \hat y(x) = w_0 + \sum_{i=1}^n w_i x_i + \sum_{i=1}^{n-1} \sum_{j=i+1}^n \langle \mathbf v_i, \mathbf v_j\rangle x_i x_j \] 其中\(\langle\cdot,\cdot\rangle\)代表两个向量的内积。每个参数\(w_{ij}=\langle \mathbf v_i,\mathbf v_j\rangle\)，\(\mathbf v_i\)代表第\(i\)特征的隐向量，隐向量长度\(k\ll n\)。

以前\(w_{ij}\)和\(w_{ik}\)之间无关，但现在\(w_{ij}=\langle \mathbf v_i,\mathbf v_j\rangle\)，\(w_{ik}=\langle \mathbf v_i, \mathbf v_k\rangle\)，两者之间有共同的\(\mathbf v_i\)，也就是说所有包含特征\(x_i\)和\(x_j\)的非零特征组合都可用来学习隐向量\(\mathbf v_i\)

An Efficient Optimization Criterion

其实公式的第二项\(\sum_{i=1}^n\sum_{i=1}^n \langle \mathbf v_i, \mathbf v_j\rangle x_i x_j\)，我们可以进行一些等价变换使得其计算更加高效，毕竟上线的场景中模型推断的速度也是十分重要的。 \[ \begin{align} \sum_{i=1}^{n-1} \sum_{j=i+1}^n \langle \mathbf v_i, \mathbf v_j\rangle x_i x_j &= \frac{1}{2}\cdot \sum_{f=1}^k\left\{\left(\sum_{i=1}^n v_{if} x_i\right)^2-\sum_{i=1}^n v_{if}^2 x_i^2\right\} \end{align} \] 综合起来我们可以得到 \[ \begin{align} \hat y(x) &= w_0 + \sum_{i=1}^n w_i x_i + \sum_{i=1}^{n-1} \sum_{j=i+1}^n \langle \mathbf v_i, \mathbf v_j\rangle x_i x_j\\ &= w_0 + \sum_{i=1}^n w_i x_i +\frac{1}{2}\cdot \sum_{f=1}^k\left\{\left(\sum_{i=1}^n v_{if} x_i\right)^2-\sum_{i=1}^n v_{if}^2 x_i^2\right\} \end{align} \] 之前，计算\(\hat y\)的复杂度是\(O(kn^2)\)，改写后复杂度降低为\(O(kn)\)。

Optimization

FM可以用梯度下降来解决

首先对\(w_0\)求导： \[ \frac{\partial y}{\partial w_0} = 1 \] 对\(w_i\)求导也很简单： \[ \frac{\partial y}{\partial w_i} = x_i \] 接下来是对\(\mathbf v_{if}\)求导： \[ \frac{\partial y}{\partial \mathbf v_{if}} = x_i \sum_{j=1}^n v_{jf}x_j-x_i^2 v_{if} \]

Summary

优点：

  • 考虑了

缺点：

  • 不同类型的特征交叉没有区分性

Field-aware Factorization Machine

Field-aware Factorization Machine（简称FFM）

Field-aware Feature Crossing

FFM \[ \hat y(x) = w_0 + \sum_{i=1}^n w_i x_i + \sum_{i=1}^{n-1} \sum_{j=i+1}^n \langle \mathbf v_{i,f_j}, \mathbf v_{j, f_j}\rangle x_i x_j \] FFM模型公式和FM的不同之处仅在于二阶交叉时的隐向量。在FFM中，每个特征有\(F\)个隐向量，\(F\)为filed的数量，也可以理解为特征分组的数量。

Optimization

模型的表达式为： \[ \hat y(x) = w_0 + \sum_{i=1}^n w_i x_i + \sum_{i=1}^{n-1} \sum_{j=i+1}^n \sum_{q=1}^k \mathbf v_{i,f_j}^q \mathbf v_{j, f_j}^q x_i x_j \]

Summary

优点：

  • 相对于FM

缺点：

  • 时间开销大
